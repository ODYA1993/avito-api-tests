## После удаления объявления статистика остаётся доступной
| Поле | Значение |
|------|----------|
| **Краткое описание** | При удалении объявления через DELETE /api/2/item/{id} статистика по этому объявлению продолжает возвращаться при GET-запросе |
| **baseUrl** | `https://qa-internship.avito.com` |
| **Приоритет** | High |
| **Серьёзность** | Major |
| **Предусловие** | Создано объявление с ID = X |
| **Шаги воспроизведения** | 1. DELETE `{{baseUrl}}/api/2/item/X` <br> 2. GET `{{baseUrl}}/api/1/statistic/X` |
| **Фактический результат** | Шаг 1: код 200, объявление удалено <br> Шаг 2: код 200, статистика возвращается |
| **Ожидаемый результат** | Шаг 1: код 200, объявление удалено <br> Шаг 2: код 404, статистика не найдена |
| **Окружение** | Postman 12.4.3, API версии 1 и 2 |

## Создание объявления без поля statistics возвращает ошибку
| Поле | Значение |
|------|----------|
| **Краткое описание** | API возвращает ошибку при создании объявления без поля `statistics` |
| **Примечание** | **Требуется уточнение:** является ли поле `statistics` обязательным? <br><br> **Если обязательно:** ошибка корректна, но сообщение должно быть информативнее ("требуется поле statistics" вместо "не передано тело объявления") <br><br> **Если опционально:** это баг, должны быть дефолтные значения |
| **Серьёзность** | Уточняется (Minor если ошибка в сообщении, Major если поле опционально) |
